## 1. 문서 개요

### 1.1 목적
본 문서는 와플래시 게임 아카이브에 보존된 **팩맨(Pac-Man) 플래시 게임**(제작: Neave)의 게임플레이를, 더 이상 구동되지 않는 Flash(.swf) 환경에서 **현대 웹(HTML5 Canvas + JS)으로 재구현**하는 미니게임의 요구사항을 정의한다. 게임 메커니즘은 **아케이드 원작에 충실하게** 유지하되, **모든 그래픽/사운드 리소스는 신규 테마로 교체**한다(리소스는 추후 별도 업로드).

### 1.2 참고 자료
- 원작(아카이브): 팩맨 플래시게임 — https://vidkidz.tistory.com/120 (와플래시 게임 아카이브, 제작 Neave, `pacman.swf`, 0.04MB)
- 룰 원천: 남코 아케이드 **Pac-Man (1980)** 의 메커니즘을 기준으로 함
- 원작 핵심 조작: 키보드 방향키(←↑↓→)

### 1.3 원작과의 차이
| 구분 | 원작(아케이드/플래시) | 본 프로젝트 |
|------|------|--------------|
| 기술 | Flash(.swf) / ActionScript | HTML5 Canvas 2D + 순수 JS |
| 플랫폼 | PC (Flash Player) | **PC + 모바일** (세로형 반응형) |
| 조작 | 방향키 | **방향키 + 스와이프/가상 D-pad** |
| 진행 구조 | 동일 미로 무한 반복 | **고정 5스테이지 + 엔딩** (미로 4종 A~D) |
| 리소스 | 원작 에셋 | **신규 제작 (별도 업로드)** — 플레이스홀더로 선개발 |
| 저장 | 기기 하이스코어 | **최고기록 localStorage 저장** |

## 2. 게임 개요

### 2.1 컨셉
플레이어는 미로 안의 캐릭터(이하 **팩맨**)를 4방향으로 조작해 미로에 깔린 **도트(점)** 를 모두 먹는다. 미로에는 4마리의 **고스트(유령)** 가 각자 다른 성격(추격 패턴)으로 팩맨을 쫓는다. 네 모서리의 **파워펠릿**을 먹으면 잠시 동안 입장이 역전되어 고스트를 잡아먹을 수 있다. 도트를 모두 먹으면 스테이지를 클리어하고, **정해진 마지막 스테이지까지 클리어하면 엔딩**을 본다.

### 2.2 핵심 재미 (Core Fun)
- **쫓고 쫓기는 긴장감**: 4마리 고스트의 협공을 동선으로 피해내는 순간 판단
- **역전의 쾌감**: 파워펠릿 → 고스트 연쇄 포식(200·400·800·1600점)의 한 방
- **숙련의 보상**: 고스트별 고유 AI를 이해할수록 안전 경로/유인이 가능 — 실력 성장 곡선

### 2.3 타겟
- 한 판 수 분 내의 아케이드 감성을 즐기는 캐주얼~코어 유저
- PC 웹 / 모바일 웹 양쪽 (세로형 우선)

## 3. 기술 사양

### 3.1 기술 스택 🟢
- **HTML5 Canvas 2D + 순수 JavaScript (Vanilla JS, 외부 게임 엔진 미사용)**
- **클래식 스크립트 로드**(ES Module 미사용) → 로컬에서 `index.html` 더블클릭(`file://`)만으로 실행 가능 (워크스페이스 공통 방식)
- 빌드/배포: 정적 파일(`index.html` + JS + 에셋)로 어디서나 호스팅 가능, 의존성 최소화

### 3.2 플랫폼 / 조작 🟢 — PC + 모바일
| 플랫폼 | 이동 |
|--------|------|
| PC | **방향키 ←↑↓→** (원작 동일) |
| 모바일 | **스와이프**(상하좌우 플릭) + **화면 가상 D-pad**(좌하단) 병행 |

- 팩맨은 **그리드(타일) 기반 4방향 이동**. 입력은 "다음에 꺾을 방향"으로 **버퍼링**되어, 교차로에 도달하는 순간 가능하면 즉시 방향 전환(원작의 코너링 감각 재현). 🟡
- 모바일 스와이프와 가상 D-pad는 동시 제공(사용자가 편한 쪽 사용). 🟡 위치·민감도는 한 손 플레이 기준으로 테스트 후 조정.

### 3.3 화면 / 반응형 🟢
- **기준 디자인 해상도: 720 × 1280 (세로형 9:16)** — 워크스페이스 공통 규격
- 미로는 원작과 동일한 **28열 × 31행 타일 격자**를 기본으로 한다. 🟡 권장 렌더: 타일 한 변 **24px**(미로 672×744px)로 화면 중앙 배치, 위쪽에 점수 HUD, 아래쪽에 목숨·과일 표시 영역.
- 캔버스는 화면 비율에 맞춰 letterbox 스케일, 논리 좌표계는 720×1280 고정.
- PC: 중앙 정렬 + 좌우 여백. 모바일: 세로 풀스크린, 회전 불필요. 🟡 안전영역(노치/홈바) 패딩 확보.

### 3.4 성능 목표 🟡
- 60 FPS 유지 (저사양 모바일 30 FPS 하한). 고정 타임스텝 로직 + 가변 렌더 권장.
- 에셋 스프라이트 시트화로 드로우콜 최소화. 미로 정적 레이어는 1회 오프스크린 캐싱.

## 4. 게임플레이

### 4.1 핵심 루프
```
타이틀 → (스테이지 인트로: READY!) → [도트 먹기 ↔ 고스트 회피 ↔ 파워펠릿 역전]
   → 도트 전부 섭취 → 스테이지 클리어 → (다음 스테이지) ...
   → 마지막 스테이지 클리어 → 엔딩 → 결과(최고기록)
                              └ 목숨 모두 소진 → 게임오버 → 결과(최고기록)
```

### 4.2 게임 목표 🟢 — 전 스테이지 클리어 + 엔딩
- 각 스테이지의 미로에 깔린 **모든 도트(일반 도트 + 파워펠릿)** 를 먹으면 클리어 → 다음 스테이지.
- **고정 5스테이지**(미로 4종 A~D)를 모두 클리어하면 **엔딩 연출** 후 결과 화면.
- 스테이지가 오를수록 고스트 속도↑, 겁먹음(파워펠릿 효과) 시간↓ 등 난이도 상승(§7 데이터 테이블).

### 4.3 이동 / 미로 규칙 🟢
- 미로는 벽/길/도트/파워펠릿/고스트 집(house)/터널로 구성(§부록 C 타일맵 포맷).
- 팩맨·고스트 모두 길 타일만 이동, 벽 통과 불가.
- **터널(좌우 워프)**: 미로 양 끝의 통로로 들어가면 반대편으로 빠져나온다. 터널 구간에서 **고스트는 감속**(원작 동일). 🟡

### 4.4 도트 / 파워펠릿 🟢
- **일반 도트**: 10점. 미로 전체에 다수 배치(원작 기준 240개). 먹는 순간 팩맨이 아주 살짝 감속(원작의 dot-eating 딜레이 재현). 🟡
- **파워펠릿(에너자이저)**: 50점. 미로 네 모서리 부근 4개. 먹으면 **겁먹음 모드** 발동(§4.6).

### 4.5 고스트 / AI 🟢 — 원작 충실 (4종 고유 인격)
4마리는 각자 다른 **추격 타깃 산정 규칙**을 가진다. 상세 알고리즘은 **부록 B** 참조.

| 고스트 | 색(원작) | 별명 | 체이스(추격) 행동 요약 |
|--------|----------|------|------------------------|
| **블링키** | 빨강 | Shadow | 팩맨의 **현재 타일**을 직접 추격(가장 공격적). 도트가 일정 수 이하로 줄면 가속(**크루즈 엘로이**). |
| **핑키** | 분홍 | Speedy | 팩맨 **진행 방향 4칸 앞**을 노려 매복. 원작의 '위 방향' 오버플로 버그(위를 볼 때 4칸 위 + 4칸 왼쪽)까지 **재현**(원작 충실, 코드 토글 가능). |
| **잉키** | 청록 | Bashful | 팩맨 2칸 앞 지점을 기준으로 **블링키 위치를 2배 연장**한 지점을 타깃(변칙적). |
| **클라이드** | 주황 | Pokey | 팩맨과 **8타일 초과**면 블링키처럼 직접 추격, **8타일 이하**면 자기 코너로 후퇴(겁쟁이). |

- **모드 전환(스캐터 ↔ 체이스)**: 일정 시간마다 전 고스트가 **스캐터(각자 지정 코너로 흩어짐) ↔ 체이스(추격)** 를 번갈아 수행. 스테이지가 오를수록 스캐터 시간이 짧아진다(부록 B 표). 모드 전환 시 고스트는 **즉시 반대 방향으로 전환**.
- **스캐터 코너**: 블링키=우상, 핑키=좌상, 잉키=우하, 클라이드=좌하.
- **고스트 집(House) 출발 규칙**: 게임 시작 시 블링키만 미로에, 나머지는 집 안에서 대기. **도트 카운터**가 임계치에 도달할 때마다 핑키→잉키→클라이드 순으로 출발(부록 B). 목숨 소진 후 재시작 시 글로벌 도트 카운터 규칙 적용.
- 교차로에서 고스트는 **현재 타깃 타일에 가장 가까워지는 방향**을 선택하되, **방금 온 방향으로의 즉시 역주행은 금지**(모드 전환·겁먹음 발동 시 예외).

### 4.6 겁먹음(Frightened) 모드 🟢 — 파워펠릿 역전
- 팩맨이 파워펠릿을 먹으면 일정 시간 동안 전 고스트가 **파랗게 질려** 감속하고, 교차로에서 **무작위(의사난수) 방향**으로 도망친다. 발동 즉시 고스트는 **반대 방향으로 전환**.
- 이 동안 팩맨은 고스트를 **잡아먹을 수 있다**. 한 번의 파워펠릿으로 연쇄로 먹을 때 **200 → 400 → 800 → 1600점** 으로 증가.
- 먹힌 고스트는 **눈(eyes)** 만 남아 집으로 빠르게 복귀 후 부활.
- 겁먹음 **지속 시간/종료 점멸 횟수는 스테이지가 오를수록 감소**, 후반 스테이지에서는 **0초(겁먹음 없음, 단 방향 반전만)** 가 될 수 있다(부록 B 표). 🟡

### 4.7 과일(보너스) 🟢
- 스테이지 도중 미로 중앙(고스트 집 아래)에 **과일 보너스**가 2회 등장(원작: 도트 70개·170개 섭취 시점). 일정 시간(약 9~10초) 후 사라짐. 🟡
- 과일 종류/점수는 스테이지별 상이(체리100 → 딸기300 → 오렌지500 → 사과700 → 멜론1000 …, §7). 신규 테마에서는 과일을 테마 소품으로 대체 가능.

### 4.8 목숨 / 추가 목숨 🟢 — 원작식
- 시작 목숨 **3개**(잔여 표시는 HUD 하단).
- 고스트(겁먹지 않은 상태)와 접촉 시 **목숨 -1**, 짧은 사망 연출 후 팩맨·고스트가 시작 위치로 리셋(READY!).
- **추가 목숨**: 누적 **10,000점** 도달 시 +1(원작 기본값, 🟡 임계치 조정 가능).
- 목숨 0에서 추가 사망 시 **게임오버**.

### 4.9 점수 / 최고기록 🟢
| 항목 | 점수 |
|------|------|
| 일반 도트 | 10 |
| 파워펠릿 | 50 |
| 고스트 포식(연쇄) | 200 / 400 / 800 / 1600 |
| 과일 | 100~5000 (스테이지별, §7) |
- **최고기록(High Score)** 은 `localStorage`에 저장하여 재방문 시 유지. HUD 상단과 결과 화면에 표시.
- 별도 서버/온라인 랭킹 연동은 범위 외. 🟢

### 4.10 클리어 / 게임오버 조건 🟢
- **스테이지 클리어**: 해당 미로의 모든 도트+파워펠릿 섭취.
- **엔딩**: 마지막(5번째) 스테이지 클리어.
- **게임오버**: 목숨을 모두 소진.

## 5. 화면 구성 / UI

### 5.1 화면 흐름
```
[타이틀] → [튜토리얼(최초 1회/선택)] → [스테이지 인트로 READY!] → [인게임]
   → [스테이지 클리어] → 다음 스테이지 / (마지막 후) [엔딩] → [게임 결과] → 타이틀
                                                      └ 목숨 소진 → 게임오버 → 결과
```

### 5.2 화면별 요소 (세로형 720×1280 기준)
- **타이틀**: 로고, 시작 버튼, 사운드 토글, 최고기록 표시
- **인게임 HUD**:
  - 상단: `1UP` 현재 점수 / `HIGH SCORE` 최고기록 (원작 레이아웃 오마주)
  - 중앙: 미로(28×31) + 팩맨 + 고스트 4 + 도트/파워펠릿 + 과일
  - 하단: 잔여 목숨 아이콘, 최근 등장 과일 아이콘. 모바일은 좌하단 가상 D-pad 오버레이
- **READY! 인트로**: 미로 표시 + "READY!" 텍스트 + 짧은 시작 징글
- **스테이지 클리어**: 미로 점멸 연출 후 다음 스테이지 전환
- **엔딩**: 마지막 스테이지 클리어 축하 연출
- **게임 결과**: 총점 / 최고기록(갱신 시 강조) / 다시하기 · 타이틀 버튼

### 5.3 연출 🟡
- 팩맨: 입을 벌렸다 닫는 방향별 4프레임 애니메이션, 사망 시 회전·소멸 애니
- 고스트: 이동(눈동자가 진행 방향), 겁먹음(파랑 → 종료 직전 흰색 점멸), 눈(eyes) 복귀 상태
- 도트 섭취/과일/고스트 포식/추가목숨/사망 각각의 SFX + 인게임 사이렌 BGM(겁먹음 시 변주)

## 6. 리소스(에셋) 명세

## 부록 A. 리소스 명명 규칙 & 규격 가이드 🟢

신규 에셋을 매끄럽게 교체할 수 있도록 아래 규칙을 권장한다(코드에서 동일 키로 로드).

### A.1 폴더 구조
```
assets/
├─ pacman/      팩맨 스프라이트(이동/사망)
├─ ghost/       고스트 4종 + 겁먹음/눈
├─ maze/        벽/도트/펠릿/집/문/터널 타일
├─ fruit/       과일·보너스
├─ effect/      점수 팝업·연출
├─ background/  배경
├─ ui/          로고·버튼·목숨아이콘·D-pad
├─ sfx/         효과음
└─ bgm/         배경음악(사이렌 등)
```

### A.2 명명 규칙
- **소문자 + 스네이크 케이스**, 형식: `분류_이름_상태_방향_번호`
- 연속 프레임은 2자리 0패딩: `_01`, `_02` …
- 스프라이트 시트는 `*_sheet.png` + 동명 `*_sheet.json`(프레임 좌표) 권장

| 분류 | 예시 파일명 |
|------|-------------|
| 팩맨 | `pac_move_right_01.png`~`_03.png`(상하좌우), `pac_death_01.png`~`_11.png` |
| 고스트 | `ghost_blinky_right_01.png` …, `ghost_frightened_01.png`, `ghost_flash_01.png`, `ghost_eyes_right.png` |
| 미로 | `maze_wall.png`(또는 코너셋), `maze_dot.png`, `maze_pellet.png`, `maze_door.png`, `maze_tunnel.png` |
| 과일 | `fruit_cherry.png`, `fruit_strawberry.png`, `fruit_orange.png`, `fruit_apple.png`, `fruit_melon.png` … |
| UI | `ui_logo.png`, `ui_btn_start.png`, `ui_life.png`, `ui_btn_sound.png`, `ui_dpad.png` |
| 효과음 | `sfx_dot.mp3`, `sfx_pellet.mp3`, `sfx_eat_ghost.mp3`, `sfx_fruit.mp3`, `sfx_death.mp3`, `sfx_extra.mp3` |
| 배경음 | `bgm_siren.mp3`, `bgm_frightened.mp3`, `jingle_start.mp3` |

### A.3 규격 권장
- **기준 해상도**: 720×1280(@1x). 선명도를 위해 **2x 제작 후 다운스케일** 권장
- **타일 기준**: 미로 타일 한 변 1x **24px** 권장(28×31 격자 = 672×744px). 캐릭터/고스트는 타일보다 약간 크게(코너링 시 자연스럽게).
- **이미지 포맷**: PNG, 투명 배경(32bit). 사진성 배경만 JPG 허용
- **팩맨/고스트**: 방향별 프레임 균등 크기·동일 앵커. 고스트 겁먹음/눈 상태 별도 프레임
- **사운드**: `mp3`(또는 `ogg`), SFX 모노 / BGM 스테레오, -14 LUFS 내외 노멀라이즈
- **스프라이트 시트**: 프레임 균등 간격, 여백 2px

> 사내 기존 네이밍/규격 규칙이 있으면 알려주시면 본 가이드를 그에 맞춰 갱신합니다.

## 부록 B. 고스트 AI 상세 (구현 레퍼런스) 🟢

> 원작(남코 1980) 동작 기준. 좌표는 **타일 단위**, 미로 좌상단이 (col=0,row=0).

### B.1 타깃 타일 산정 (체이스 모드)
```
blinky_target = pac_tile                                  // 팩맨 현재 타일
pinky_target  = pac_tile + 4 * pac_dir                    // 진행 방향 4칸 앞
                // 🟡 원작 'UP' 버그: dir==UP이면 (+4 up, +4 left)도 함께 — 재현 여부 결정
inky_target   = blinky_tile + 2 * ( (pac_tile + 2*pac_dir) - blinky_tile )
                // 팩맨 2칸 앞 지점을 기준점으로, 블링키→기준점 벡터를 2배 연장
clyde_target  = (distance(clyde_tile, pac_tile) > 8) ? pac_tile : clyde_scatter_corner
```
- **스캐터 모드**: 각 고스트는 자기 코너(블링키 우상 / 핑키 좌상 / 잉키 우하 / 클라이드 좌하)를 타깃.
- **이동 선택**: 다음 교차로에서, 진입 가능한 방향 중 **타깃 타일과 직선거리가 최소가 되는 방향** 선택. **직전 방향의 역주행 금지**(모드 전환·겁먹음 시작 시에는 강제 반전 허용).
- **겁먹음 모드**: 교차로마다 의사난수로 방향 선택(역주행 금지 규칙은 유지).

### B.2 스캐터 ↔ 체이스 웨이브 타이밍 (초) 🟡
| 스테이지군 | 웨이브 (S=스캐터, C=체이스) |
|------------|------------------------------|
| 1 | S7 → C20 → S7 → C20 → S5 → C20 → S5 → C(무한) |
| 2~4 | S7 → C20 → S7 → C20 → S5 → C(매우 김) → S(짧음) → C(무한) |
| 5+ | S5 → C20 → S5 → C20 → S5 → C(매우 김) → S(짧음) → C(무한) |
- 팩맨이 죽으면 현재 미로의 웨이브 타이머는 리셋.

### B.3 고스트 집 출발(도트 카운터) 🟡
- 스테이지 시작 시: 블링키는 미로 밖, 핑키·잉키·클라이드는 집 안.
- 개별 도트 카운터 임계치 도달 시 출발 — 예: **스테이지1** 핑키 0, 잉키 30, 클라이드 60 / **스테이지2** 잉키 0, 클라이드 50 / **스테이지3+** 전원 0.
- 팩맨 사망 후 재시작 시에는 **글로벌 도트 카운터**(예: 핑키 7, 잉키 17, 클라이드 32)로 출발 제어. 일정 시간 도트를 못 먹으면 강제 1마리 출발(원작의 4초 타이머).

### B.4 크루즈 엘로이 (블링키 가속) 🟡
- 미로의 잔여 도트가 임계치(§7 표) 이하로 떨어지면 블링키가 팩맨 속도 이상으로 가속(엘로이1), 더 줄면 추가 가속(엘로이2). 스테이지가 오를수록 발동 임계/가속 폭 증가.

### B.5 속도 환산 기준 🟡
- "100% = 기준 최고 속도"를 px/초로 1개 상수로 정의하고, 표의 %를 곱해 각 개체 속도를 산출. 터널 진입 시 고스트는 표의 터널 % 적용. 프레임 독립(델타타임) 이동 권장.

## 부록 C. 미로(타일맵) 포맷 정의 🟢

미로는 텍스트(또는 JSON) 격자로 정의하여 **데이터/코드 분리**. 28열 × 31행 기본.

### C.1 타일 범례
| 기호 | 의미 |
|------|------|

| `P` | 팩맨 시작 위치 |
| `b` `p` `i` `c` | 고스트 시작 위치(블링키/핑키/잉키/클라이드) |

```
T............    ............T   ← 좌우 터널 워프
... (중략) ...
#o..##................##..o#
############################
```
