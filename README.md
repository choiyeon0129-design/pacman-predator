# 포식자 — Pac-Man 리메이크

미로에서 도트를 먹고 4마리 고스트를 피하는 아케이드 미니게임. HTML5 Canvas + 순수 JavaScript.
원작(와플래시 아카이브의 팩맨 플래시게임)의 메커니즘을 **원작 충실**하게 재구현. 게임 제목은 **포식자**. 적은 데몬 스프라이트(`demon_1`/`demon_2`), 배경은 맵 이미지(`map.png`).

- 원작 참고: https://vidkidz.tistory.com/120 (Flash, 구동 불가 → 웹으로 재구현)
- 기획: [팩맨_기획_명세서.md](팩맨_기획_명세서.md)
- 현재 단계: **M1–M3 구현 완료 — 플레이 가능**. 폰트 **Puzzle Sans**. 게임 제목 **'포식자'** + **시작하기 버튼**. 적 = **픽셀 데몬 2종**(`demon_1`/`demon_2`, 4마리에 배분), 팩맨 = 골드 스프라이트. **배경(게임 맵) = `map2.png`(전 화면 공통)**, 미로 = **평면 라인**(#4ca5a8). 게임오버/엔딩은 미로 없이 배경만. (미로 레이아웃 1종 사용)
- 레이아웃: **PC는 화면 D-pad 없이** 빈 공간을 줄인 캔버스(720×928), **모바일/터치는 가상 D-pad** 포함(720×1220). 캔버스는 창 크기에 맞춰 반응형 스케일. (기기 판별: `applyLayout()` — pointer:coarse)

## 확정된 핵심 사양 (요약)
| 항목 | 결정 |
|------|------|
| 플랫폼/조작 | PC(방향키/WASD) + 모바일(스와이프 + 가상 D-pad) |
| 고스트 AI | 원작 충실 — 4종 고유 AI, 스캐터/체이스, 파워펠릿/과일/터널 |
| 진행 구조 | 고정 5스테이지(미로 A,B,C,D,A) + 엔딩 |
| 목숨/점수 | 목숨 3 + 추가목숨(10,000점) + 최고기록 localStorage |

## 실행 방법
- 가장 간단: `index.html` **더블클릭** (브라우저에서 바로 실행 — 클래식 스크립트라 `file://` 지원)
- 로컬 서버(권장 · node/python 불필요):
  ```
  powershell -ExecutionPolicy Bypass -File serve.ps1
  # http://localhost:8120 접속
  ```
  `serve.ps1` 은 PowerShell 내장 HTTP 서버(의존성 없음). 포트 변경: `-Port 9000`

## 조작
| 플랫폼 | 이동 | 화면 전환 |
|--------|------|-----------|
| PC | 방향키 ←↑↓→ 또는 WASD | 스페이스 / 엔터 |
| 모바일 | 스와이프(상하좌우) + 좌하단 가상 D-pad | 화면 탭 |

## 규칙 (요약)
- 미로의 모든 도트 + 파워펠릿을 먹으면 스테이지 클리어 → 다음 스테이지
- 파워펠릿을 먹으면 일정 시간 고스트를 잡아먹을 수 있음(연쇄 200·400·800·1600점)
- 고스트(겁먹지 않은 상태) 접촉 시 목숨 -1, 목숨 모두 소진 시 게임오버
- 마지막(5번째) 스테이지 클리어 시 엔딩
- 자세한 룰/AI/밸런싱: 기획 명세서 §4·§7 및 부록 B 참조

## 구현 현황
- ✅ 미로/도트/펠릿(244개) · 팩맨 이동·코너링(중심도달 모델)·터널 워프
- ✅ 고스트 4종 고유 AI(블링키/핑키/잉키/클라이드) + 스캐터/체이스 전환 + 집 출발(도트 카운터·강제 타이머)
- ✅ 파워펠릿 겁먹음(반전·감속·랜덤) + 연쇄 포식 + 눈(eyes) 집 복귀/부활
- ✅ 핑키 '위 방향' 오버플로 버그 재현(`PINKY_UP_BUG` 토글)
- ✅ 목숨 3 + 추가목숨(10,000점) + 점수/최고기록(localStorage)
- ✅ 5스테이지 난이도 테이블 + 미로 교체 흐름 + 과일 2회 등장 + 타이틀/READY/클리어/엔딩/게임오버
- 🟡 **미로 B/C/D는 현재 미로 A 재사용** — 시스템은 N미로 지원(`js/config.js`의 `MAZE_B/C/D` 문자열만 교체)
- 🟡 사운드는 경량 WebAudio 플레이스홀더

## 리소스 교체 워크플로
적(데몬)은 투명 픽셀 PNG `demon_1.png`/`demon_2.png`(루트)를 4마리에 배분(`GHOST_DEMON`, config.js, 종횡비 유지)해 그대로 그리고, 팩맨은 `assets/pacman.png`(진행 방향 회전 + 원형 클리핑). 게임 맵(배경)은 `map.png`(약간 어둡게). 텍스트 폰트는 `PuzzleSans.ttf`(원본 `폰트.ttf`의 ASCII 복사본, @font-face — 가족명 'Puzzle Sans'). 로더는 `js/assets.js`이며 로드 실패 시 **벡터/단색 폴백**.
- 적(데몬) 교체: 루트 `demon_1.png`/`demon_2.png` 덮어쓰기(매핑은 `GHOST_DEMON`). 팩맨: `assets/pacman.png`.
- 게임 맵(배경): 루트 `map.png`(키 `map`, `drawBackground()` cover-fit + 약간 어둡게). 로드 실패 시 `COLOR.bg`(#0c1420) 단색 폴백. (`map1.png`/`thumnail.png`·`assets/ghost_*`는 미사용·보존)
- 과일·이펙트 등 추가 에셋은 기획 명세서 **부록 A** 명명 규칙 권장.
- 겁먹음(파랑)·눈(eyes) 상태는 의도적으로 벡터로 그려 가독성을 유지한다.
